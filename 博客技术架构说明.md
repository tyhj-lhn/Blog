# MemoryStory Blog — 技术架构说明

> 一个用 AI 辅助构建的个人博客，从数据库到 UI 交互的完整技术拆解。

---

## 项目概览

MemoryStory 是一个设计风格走 Swiss Modernism 路线的个人博客，核心体验包括：

- 全屏 hero 动态壁纸（视频/图片），透明导航栏浮动其上
- Markdown 博文 + 盖楼式嵌套评论（类似论坛回帖）
- 隐藏式后台管理（无公开登录入口，仅通过 `/admin` 路径直达）
- 完整的 CMS：文章编辑器、壁纸管理、评论管理、关于页编辑

**技术标签：** `React 19` · `TypeScript` · `Fastify 5` · `Prisma 6` · `PostgreSQL 17` · `Tailwind CSS 4` · `PM2`

### 请求流

```
浏览器 → Nginx → /api/* → Fastify (:3001) → Prisma → PostgreSQL
              → /*      → 静态 SPA 文件 (前端路由由 React Router 接管)
```

Nginx 把 `/api` 请求反代到后端，其余路径返回 SPA 的 `index.html`。后端只监听 `127.0.0.1:3001`，不对外暴露。

### 关键前端机制

#### 401 刷新队列

多个 API 请求同时 401 → 只发一次 refresh → 所有请求排队等待同一 Promise：

```typescript
// api.ts 核心逻辑（简化）
let refreshPromise: Promise<string | null> | null = null;

function refreshAccessToken(): Promise<string | null> {
  if (!refreshPromise) {
    refreshPromise = fetch('/api/auth/refresh', {
      method: 'POST',
      body: JSON.stringify({ refreshToken: getRefreshToken() }),
    })
      .then(r => r.ok ? r.json() : null)
      .then(data => {
        if (data?.accessToken) setAccessToken(data.accessToken);
        return data?.accessToken ?? null;
      })
      .finally(() => { refreshPromise = null; });
  }
  return refreshPromise;
}
```

#### 乐观更新

**点赞：** 点击心 → 立即更新 UI（`liked` 状态 + `likeCount ±1`）→ 后台发 `POST /api/posts/:slug/toggle-like` → 失败回滚。`localStorage` 存已点赞 post ID 集合做去重，`liked` 渲染时从 localStorage 派生，保证跨页面一致性。

**评论：** 提交评论 → `onMutate` 乐观 +1 `_count.comments` → `onError` 回滚 → `onSettled` 刷新 post + 评论树。

### buildApp() 工厂 + PM2 桥接

```typescript
// index.ts
export function buildApp() {
  const app = Fastify({ logger: true, bodyLimit: 512 * 1024 });
  // 注册插件、路由...
  return app;
}

export async function startServer() {
  const app = buildApp();
  await prisma.$connect();       // 数据库连接（重试 2 次，间隔 3s）
  validateSecrets();             // 生产模式拒绝弱密钥
  await app.listen({ port: 3001, host: '127.0.0.1' });
  process.send?.('ready');       // PM2 wait_ready 信号
}
```

PM2 7.x fork 模式不兼容 `"type": "module"` 的 `.js` 文件 → CJS 启动器桥接：

```javascript
// boot.cjs
(async () => {
  const mod = await import('./dist/index.js');
  await mod.startServer();
})();
```

**评论树查询** — PostgreSQL 递归 CTE：

```sql
WITH RECURSIVE comment_tree AS (
  SELECT *, 0 AS depth FROM "Comment"
  WHERE "postId" = $1 AND "parentId" IS NULL
  UNION ALL
  SELECT c.*, ct.depth + 1 FROM "Comment" c
  JOIN comment_tree ct ON c."parentId" = ct.id
)
SELECT * FROM comment_tree ORDER BY depth, "createdAt";
```

后端 `buildCommentTree()`：第一遍 `Map<id, node>`，第二遍分配到 `roots[]` 或 `parent.children[]`。前端 `CommentTree` 递归渲染，每层不同颜色左边框。

## 安全设计

### 认证

```
登录 → accessToken (15min) + refreshToken (7d)
请求 → Authorization: Bearer <accessToken>
401  → 前端自动刷新队列换新 token
改密 → tokenVersion 原子递增 → 所有旧 token 立即失效
```

### XSS 防护

`sanitize.ts` — `xss` 库空白名单：**所有** HTML 标签、属性、事件处理器全部过滤。评论内容、用户名、留言昵称、留言内容在写入 DB 前统一经过 `sanitizeContent()`。

### 登录锁定

`login-guard.ts` 内存 Map 追踪：

| 失败次数 | 锁定时间 |
|---------|---------|
| 5 | 15 分钟 |
| 10 | 1 小时 |

30 分钟无失败自动重置。锁定检查在 DB 查询前。"用户不存在"和"密码错误"返回相同响应。

### 文件上传安全

| 措施 | 实现 |
|------|------|
| 扩展名 | 仅 `.jpg` `.jpeg` `.png` `.mp4` |
| MIME | `file.mimetype` 白名单校验 |
| 文件名 | UUID 替代原始名 |
| 大小 | 图片 5MB / 视频 50MB |
| 路径遍历 | 拒绝 `..` 和 `/` |
| 访问控制 | JWT authGuard |

### 速率限制

| 端点 | 限制 |
|------|------|
| 全局 | 100 req/min |
| 登录 | 5 req/min |
| 评论/留言 | 3 req/min |
| 改密 | 5 req/min |

## 部署架构

### 服务器

- **OS：** Ubuntu 22.04（阿里云 ECS）
- **进程：** PM2 fork 模式，`boot.cjs` 入口，`wait_ready: true`
- **反代：** Nginx（SPA try_files + `/api` + `/uploads` 反代 + gzip）
- **SSL：** Let's Encrypt（certbot 自动续期）
- **数据库：** PostgreSQL 17（仅 localhost）

### Nginx 核心

```nginx
server {
    listen 443 ssl;
    root /var/www/memorystory/frontend/dist;

    location / { try_files $uri $uri/ /index.html; }    # SPA fallback
    location /api/ { proxy_pass http://127.0.0.1:3001; } # API 反代
    location /uploads/ { proxy_pass http://127.0.0.1:3001; }

    gzip on;
    gzip_types text/css application/json application/javascript text/xml;
}
```

### 部署脚本

```bash
bash setup-server.sh   # 首次环境安装（6 步：基础包 → Node → PG → Nginx → PM2 → ufw）
bash deploy-app.sh     # 网站部署/更新（8 步：克隆 → .env → 构建 → PM2 → Nginx → SSL → 验证）
```
